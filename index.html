Step 1

Telegram Bot Banana

BotFather se apna bot create karo

💬

BotFather kya hai?

BotFather Telegram ka official bot hai jo naye bots banata hai. Iske bina koi Telegram bot nahi ban sakta.

🎯

Hume kya milega?

Ek BOT_TOKEN milega — ye aapke bot ka password hai. Ise kisi ke saath share mat karna.

1
Telegram kholo aur @BotFather search karo
2
/newbot command bhejo
3
Bot ka naam likho — e.g. "JT GitHub Notify Bot"
4
Username likho — e.g. JTGithubNotifyBot (end mein "bot" hona chahiye)
5
Token copy karo aur safe jagah rakh lo!
Token Format
7123456789:AAHdqTcvCH1vGWJxfSeofSs0K5PALDsaw
# Yeh sirf example hai — aapka apna hoga
⚠️ BOT_TOKEN kisi ke saath share mat karo — ye aapke bot ka master password hai!
🔗 t.me/BotFather kholna

Step 2

Telegram API Credentials

API_ID aur API_HASH lena — bilkul free

🔑

Kyun chahiye?

Bot Telethon library use karta hai jo Telegram ke MTProto protocol se baat karta hai. Iske liye API credentials zaroori hain.

✅

Kya milega?

API_ID (number) aur API_HASH (string) — dono bilkul free hain.

1
Browser mein jao: my.telegram.org
2
Apne Telegram phone number se login karo
3
"API Development Tools" par click karo
4
Form bharo: App title = "JT Bot", Short name = "jtbot"
5
"Create Application" click karo
6
API_ID aur API_HASH copy karo
Credentials Example
API_ID=12345678        # Sirf numbers
API_HASH=abcdef1234567890abcdef1234567890  # Long string
💡 Tip: API_ID sirf numbers ka hoga, API_HASH letters+numbers ka mix.
🔗 my.telegram.org

Step 3

MongoDB Atlas Database

Free database banana — koi credit card nahi chahiye

🍃

MongoDB Atlas kya hai?

Ye ek free cloud database hai. Bot users, tokens, aur repo data yahan store karta hai. M0 tier (free) kaafi hai is bot ke liye.

1
mongodb.com/atlas par jao aur free account banao
2
"Build a Database" click karo
3
"Free" (M0) tier choose karo — AWS ya Google Cloud
4
Username aur password set karo — yaad rakh lena!
5
Network Access mein: "Allow Access from Anywhere" (0.0.0.0/0) add karo
6
Database > Connect > "Connect your application" > Connection string copy karo
MONGO_URI Format
mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/jtbot?retryWrites=true&w=majority
# username aur password apne set kiye hue daalo
# 'jtbot' database ka naam hai — kuch bhi rakh sakte ho
⚠️ Network Access mein 0.0.0.0/0 add karna mat bhoolna! Warna hosting se connection nahi hogi.
🔗 MongoDB Atlas (Free)

Step 4

GitHub OAuth App

GitHub se login ke liye OAuth App banana

🔐

OAuth kya karta hai?

Jab users /connect karte hain, GitHub se securely login hote hain. CLIENT_ID aur CLIENT_SECRET iske liye chahiye.

🔄

Callback URL?

Ye woh URL hai jahan GitHub login ke baad user wapas aata hai. Hosting deploy karne ke baad update karna hoga.

1
GitHub > Settings > Developer Settings par jao
2
OAuth Apps > New OAuth App click karo
3
Application name: "JT GitHub Bot"
4
Homepage URL: https://your-app.railway.app
5
Callback URL: https://your-app.railway.app/oauth/callback
6
"Register application" click karo
7
CLIENT_ID copy karo, phir "Generate a new client secret" click karo
GitHub OAuth Credentials
GITHUB_CLIENT_ID=Ov23liXXXXXXXXXX
GITHUB_CLIENT_SECRET=abcdef1234567890abcdef1234567890XXXXXXXX
💡 Tip: Hosting URL deploy ke baad milegi — pehle placeholder daalo, baad mein update karo.
🔗 GitHub Developer Settings

Webhook Setup

GitHub se real-time notifications ke liye

🔔

Webhook kya karta hai?

Jab bhi aapke GitHub repo mein kuch hota hai (push, PR, issue, release), GitHub aapki bot ke server ko ping karta hai — aur bot Telegram par notification bhejta hai.

🤖 Automatic (Recommended)

Bot /addwebhook command se automatically webhook set karta hai. Koi extra kaam nahi!

1
/repos bhejo
2
Repo select karo
3
/addwebhook press karo
4
Done! ✅

🔧 Manual (Agar bot se na ho)

1
GitHub > Repo > Settings > Webhooks
2
"Add webhook" click karo
3
URL: https://your-app.railway.app/webhook
4
Content type: application/json
5
"Send me everything" choose karo

Common Errors & Fixes

Koi masla? Yahan solution dhundo

❓ Bot respond nahi kar raha
  • BOT_TOKEN sahi check karo — copy paste mein galti to nahi?
  • Railway/Render logs mein koi error dekho
  • Koi variable missing to nahi? Sab ZARORI variables daale hain?
  • Deployment successful hua? Deployment status green hai?
❓ Database connection fail ho raha hai
  • MONGO_URI mein username:password sahi hai?
  • Network Access mein 0.0.0.0/0 add hai MongoDB Atlas par?
  • URI format sahi hai? (mongodb+srv://... se start hona chahiye)
  • Database name URI mein hai? (last slash ke baad likho)
❓ GitHub OAuth kaam nahi kar raha
  • BASE_URL mein https:// lagaya aur end mein / nahi?
  • GitHub OAuth App ki Callback URL exactly match karti hai BASE_URL + /oauth/callback?
  • CLIENT_ID aur CLIENT_SECRET sahi copy kiye?
  • OAuth App ko "active" status mein check karo
❓ Render par app slow / timeout
  • Render free tier mein app 15 min baad sleep ho jati hai
  • UptimeRobot.com (free) se har 5 min ping karo
  • Railway ya Koyeb try karo — better performance hai
❓ Webhook notifications nahi aa rahe
  • BASE_URL sahi hai? Hosting live aur accessible hai?
  • GitHub repo mein webhook add hua? Settings > Webhooks check karo
  • Webhook recent deliveries mein error to nahi?
  • /addwebhook dobara chalao
💡 Pro Tip: Hamesha hosting platform ke "Logs" section mein dekho — wahan exact error milti hai jo problem solve karna aasaan banati hai!
